### 2. Addressing Your Open Questions

#### Q1: Member Limits (Free vs. Pro)

- **My Recommendation**: **YES**, stick to the tier-based limits.
- **Reason**: It prevents "runaway" infrastructure costs on free accounts. 5 members for Free / 15 for Pro is a good starting point.
- **Implementation Note**: The limit should be checked at the moment of **Invite/Add**, not just creation.

#### Q2: Message Scrubbing (The "Mixed User" Problem)

- **The Problem**: In a group with both Free and Pro users, who gets scrubbed?
- **Solution**: **Scrubbing should be Per-User.**
  - If a **Free User** logs in, they only see the last 7 days of the group history.
  - If a **Pro User** in the _same group_ logs in, they see the full history.
- **Why?**: This creates "FOMO" (Fear Of Missing Out). If a Free user sees they are missing context that their Pro teammates have, they are much more likely to upgrade.

#### Q3: Privacy & Discovery

- **Recommendation**: Implement this via a `isPublic` flag in the `Chat` model.
- **Public Groups**: Should be indexed in a new "Explore" or "Search" API.
- **Private Groups**: Only accessible via a unique `inviteCode` or direct admin add.

#### Q4: "Member Tags" (The 30-Char Label)

- **Implementation**: This should be stored in the `Chat` model's `participants` array as an object: `{ userId: string, role: string, tag: string }`.
- **Pro**: It adds a "Pro" feel to the groups (e.g., "Support Agent", "Moderator").

### 3. Structural Change: The "System Message"

For this to work reliably, we need a new **Message Kind**.
Currently, your messages are mostly text. We should add a `kind` field to the `Message` model:

1.  `text`: Standard user message.
2.  `system`: Generated by the server (e.g., "X joined"). These should likely be **non-editable** and **non-deletable** by users.

### 4. Group Deletion Logic

- **Refinement**: I recommend allowing the Owner to delete the group **even if there are members**, but it must be a "Hard Delete" (removes the chat for everyone).
- **Alternative**: "Archive" mode where the group becomes read-only for everyone but remains in history.
- **Recommendation**: confirm from user to type `Delete <Group Name>` using modal.
